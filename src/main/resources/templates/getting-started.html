Getting Started

Learn how to run and explore the Lakeside Mutual Claim Processing Application

Running the Application

Option 1: Using Maven

If you have Maven installed, you can run the application from the command line:

cd /Users/stuartwilliams/src/github.com/pidster/ddd-test-02
./mvnw spring-boot:run

Option 2: Using your IDE

If you're using an IDE like IntelliJ IDEA or Eclipse:

  1. Open the project in your IDE
  2. Find the main class ClaimProcessingApplication.java
  3. Right-click and select "Run" or "Debug"

Option 3: Using the JAR file

Build the JAR file and run it:

cd /Users/stuartwilliams/src/github.com/pidster/ddd-test-02
./mvnw clean package
java -jar target/claimprocessing-0.0.1-SNAPSHOT.jar

Accessing the Application

Once the application is running, you can access it using these URLs:

  • http://localhost:8080 - Home page
  • http://localhost:8080/dashboard - Claims dashboard
  • http://localhost:8080/h2-console - H2 Database console

For the H2 console, use these connection settings:

  • JDBC URL: jdbc:h2:mem:claimdb
  • Username: sa
  • Password: (leave empty)

Using the REST API

The application provides a RESTful API for programmatic access. Here are the main endpoints:

Policy Holders API

  • GET /api/policyholders - Get all policy holders
  • GET /api/policyholders/{id} - Get policy holder by ID
  • POST /api/policyholders - Create a new policy holder
  • PUT /api/policyholders/{id} - Update a policy holder

Claims API

  • GET /api/claims - Get all claims
  • GET /api/claims?status=SUBMITTED - Filter claims by status
  • GET /api/claims?policyHolderId={id} - Get claims for a policy holder
  • GET /api/claims/{id} - Get claim by ID
  • POST /api/claims - Create a new claim
  • PUT /api/claims/{id}/status - Update claim status

Example: Creating a new claim

curl -X POST http://localhost:8080/api/claims \
-H "Content-Type: application/json" \
-d '{
  "policyHolderId": "POLICY_HOLDER_ID",
  "policyNumber": "POL-12345",
  "description": "Vehicle damage from collision",
  "incidentDate": "2023-08-15T14:30:00",
  "claimAmount": 1250.75
}'

Project Structure

This application follows Domain-Driven Design principles with these layers:

  • Domain Layer - Core business logic
  • Application Layer - Use cases and orchestration
  • Infrastructure Layer - Technical implementations
  • Interface Layer - User interfaces

Key components:

  • Entities (Claim, PolicyHolder)
  • Value Objects (Address)
  • Domain Events
  • Repositories
  • Application Services
  • Controllers
